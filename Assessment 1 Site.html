Project Idea

Overview

The project I have in mind would be a video game that is able to change based on how the user plays. This would be a crossover between a mystery game and a visual novel. The game will have multiple paths a user can choose throughout the game, and these branches will stem from significant events that occur in-game that require more logical thinking and/or moral reasoning. These choices will also be affected by the other minor choices the player has made so far, or the player’s playstyle.


Motivation

This project would be interesting as it would give the player a personalized game experience that suits his/her behaviour and preferences. The data from the playthrough can also be used for future behavioural research, as well as for creation of an artificial intelligence system that could more accurately determine and predict human behaviour, as well as create even better games of this genre in the future.


Description

The game would use an 8-bit graphics so that the focus would be on the decision-making that is needed while playing it rather than exuberant graphics. Majority of the game would consist of events that occur in our everyday lives, and the decisions we make while doing these things would affect events in the future and the choices we are given there.

For example, while playing through the game I happen to interact with person A than person B more due to our interest in him/her. When a crime occurs that puts all suspicion on person A and person B, we would more likely have a stronger opinion on person A in our selection of responses since we had more interactions with that person. Consequently, the choices that talk about person B would have less conviction. Interacting with objects related to a crime or mystery could also help make better claims in the process of piecing things together.

The objectives of the game will be written in a virtual calendar, simulating a to-do list. The order in which the player does these tasks, as well as he does or skips them can give more information on the player’s behavioural patterns. Other interactable objects will be available in the game as well, and these would also affect the occurrences later in the game. The first playthrough would be used for the main behavioural pattern of the player as this is what seemed most natural to the player to do. Consequent playthroughs could tell us other things about the player such as curiosity or perfectionism, and the adjustments they make can also be valuable data for adaptation.


Tools and Technologies

Since the game is 8-bit, powerful hardware won’t be required to run the game so any modern-day desktop or laptop would be enough. Python would be the preferred method of coding this game due to its simplicity and readability, and the number of available libraries along with the easy learning curve. However, Java will most likely be used as I am currently more familiar with it, and it isn’t bad for creating games and AI technology as well. [1]


Skills Required

A good understanding of how decisions work and how these are affected by a person’s behavioural patterns would be needed, along with knowledge on creating a simple game program on Java.


Outcome

If the project is successful, people will be able to enjoy a customized experience that also helps in the research of human behaviour. This could lead to much better and accurate games of this type in the future, along with the development of an AI that could predict human behavioural patterns.
